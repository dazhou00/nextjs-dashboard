# 添加元数据

元数据对于 SEO 和可分享性至关重要。本章我们将讨论如何为你的 Next.js 应用添加元数据。

- 什么是元数据。
- 元数据类型。
- 如何使用元数据添加 Open Graph 图像。
- 如何使用元数据添加同档案。

## 什么是元数据？

在网页开发中，元数据提供了关于网页的额外细节。访问该页面的用户不可见元数据。相反，它在幕后工作，嵌入在页面的 HTML 中，通常是在 `<head>` 元素中。这些隐藏信息对于搜索引擎和其他需要更好地理解你网页内容的系统至关重要。

## 元数据为什么重要？

元数据在提升网页 SEO 方面起着重要作用，使其对搜索引擎和社交媒体平台更易访问和易于理解。合适的元数据帮助搜索引擎有效索引网页，提升其在搜索结果中的排名。此外，像 Open Graph 这样的元数据提升了共享链接在社交媒体上的外观，使内容对用户更具吸引力和信息量。

## 元数据类型

元数据有多种类型，每种都有其独特的用途。一些常见类型包括：

**标题元数据** ：负责浏览器标签页上显示网页标题的部分。这对 SEO 至关重要，因为它帮助搜索引擎了解网页的内容。

```html
<title>Page Title</title>
```

**描述元数据** ：这些元数据简要介绍网页内容，通常会在搜索引擎结果中显示。

```html
<meta name="description" content="A brief description of the page content." />
```

**关键词元数据** ：该元数据包含与网页内容相关的关键词，帮助搜索引擎索引页面。

```html
<meta name="keywords" content="keyword1, keyword2, keyword3" />
```

**开放图元数据** ：该元数据增强了网页在社交媒体平台上分享时的呈现方式，提供标题、描述和预览图片等信息。

```html
<meta property="og:title" content="Title Here" />
<meta property="og:description" content="Description Here" />
<meta property="og:image" content="image_url_here" />
```

**Favicon 元数据** ：该元数据将 favicon（一个小图标）链接到浏览器地址栏或标签页中显示的网页。

```html
<link rel="icon" href="path/to/favicon.ico" />
```

## 添加元数据

Next.js 有一个元数据 API，可以用来定义你的应用元数据。你可以通过两种方式向应用添加元数据：

- 基于配置 ：导出[静态`meatdata`对象](https://nextjs.org/docs/app/api-reference/functions/generate-metadata#metadata-object)或动态[生成元数据函数](https://nextjs.org/docs/app/api-reference/functions/generate-metadata#generatemetadata-function) ，在 `layout.js` 或 `page.js` 文件中。
- 基于文件的：Next.js 拥有一系列专门用于元数据目的的特殊文件：
  - `favicon.ico`、`apple-icon.jpg` 和 `icon.jpg`：用于函数和图标
  - `opengraph-image.jpg` 与 `twitter-image.jpg`：用于社交媒体图片
  - `robots.txt`：提供搜索引擎爬取的说明
  - `sitemap.xml`：提供

你可以灵活地将这些文件用作静态元数据，或者在项目内通过程序生成它们。

有了这两个选项，Next.js 会自动生成你页面相关的 `<head>` 元素。

## Favicon 和 Open Graph 图像

在你的 `/public` 文件夹里，你会注意到有两张图片：`favicon.ico` 和 `opengraph-image.jpg`。

把这些图片移到你 `/app` 文件夹的根节点。

完成后，Next.js 会自动识别并使用这些文件作为你的收藏图和原始图片。你可以在开发工具中检查应用的 `<head`> 元素来验证这一点。

知道了： 你也可以用 [`ImageResponse`](https://nextjs.org/docs/app/api-reference/functions/image-response) 构造器创建动态的原始图像。

## 页面标题与描述

你还可以包含任何 `layout.js` 或 `page.js` 文件中的元数据对象 ，以添加标题和描述等额外页面信息。`layout.js` 中的任何元数据都会被所有使用该元数据的页面继承。

在根布局中，创建一个包含以下字段的新元数据对象：

```tsx
// /app/layout.tsx

import { Metadata } from "next";

export const metadata: Metadata = {
  title: "Acme Dashboard",
  description: "The official Next.js Course Dashboard, built with App Router.",
  metadataBase: new URL("https://next-learn-dashboard.vercel.sh"),
};

export default function RootLayout() {
  // ...
}
```

Next.js 会自动将标题和元数据添加到你的申请中。

但如果你想为某个特定页面添加自定义标题呢？你可以通过在页面本身添加元数据对象来实现这一点。嵌套页面中的元数据会覆盖父页面的元数据。

例如，在 `/dashboard/invoices` 页面，你可以更新页面标题：

```tsx
// /app/dashboard/invoices/page.tsx

import { Metadata } from "next";

export const metadata: Metadata = {
  title: "Invoices | Acme Dashboard",
};
```

这个方法有效，但我们在每一页都重复应用的标题。如果有变化，比如公司名称，你就必须在每一页都更新。

相反，你可以在元数据对象中使用 `title.template` 字段来定义页面标题的模板。这个模板可以包含页面标题以及你想包含的其他信息。

在根布局中，更新元数据对象，加入模板：

```tsx
import { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    template: "%s | Acme Dashboard",
    default: "Acme Dashboard",
  },
  description: "The official Next.js Learn Dashboard built with App Router.",
  metadataBase: new URL("https://next-learn-dashboard.vercel.sh"),
};
```

模板中的 % 将被替换为具体的页面标题。

现在，在你的 `/dashboard/invoices` 页面，你可以添加页面标题：

```tsx
// /app/dashboard/invoices/page.tsx

export const metadata: Metadata = {
  title: "Invoices",
};
```

导航到 `/dashboard/invoices` 页面，检查 `<head>` 元素。你应该会看到页面标题现在是 `Invoices | Acme Dashboard` 。

Next.js 元数据 API 强大且灵活，允许您完全控制应用的元数据。这里，我们向您展示了如何添加一些基础元数据，但您也可以添加多个字段，包括 `keywords` 、 `robots` 、 `canonical`等。欢迎您自由浏览[文档](https://nextjs.org/docs/app/api-reference/functions/generate-metadata) ，并为应用添加任何额外元数据。
