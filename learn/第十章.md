## 为什么要使用 URL 搜索参数？

如上所述，你将使用 URL 搜索参数来管理搜索状态。如果你习惯用客户端状态操作，这个模式可能是新的。

实现带有 URL 参数的搜索有几个好处：

- 可收藏和共享的 URL：由于搜索参数都在 URL 中，用户可以收藏应用当前状态，包括搜索查询和筛选，以便未来参考或分享。
- 服务器端渲染 ：服务器端可以直接调用 URL 参数来渲染初始状态，从而简化服务器渲染。
- 分析与追踪 ：直接在 URL 中设置搜索查询和筛选，使得追踪用户行为更为便捷，无需额外的客户端逻辑。

## 添加搜索功能

以下是你将用来实现搜索功能的 Next.js 客户端钩子：

- `useSearchParams`——允许你访问当前 URL 的参数。例如，该 URL `/dashboard/invoices?page=1&query=pending` 的搜索参数会是这样的：`{page： '1'， query： 'pending'}`。
- `usePathname` - 让你读取当前 URL 的路径名。例如，对于路由 `/dashboard/invoices`，`usePathname` 会返回 `'/dashboard/invoices'`。
- `useRouter` - 通过程序实现客户端组件内路由之间的导航。你可以用[多种方法](https://nextjs.org/docs/app/api-reference/functions/use-router#userouter) 。

以下是实施步骤的简要概述：

1. 捕捉用户的输入。
2. 用搜索参数更新 URL。
3. 保持 URL 与输入字段同步。
4. 更新表格以反映搜索查询。

### 1. 捕捉用户输入

进入 `<Search>` 组件（`/app/ui/search.tsx`），你会注意到：

- `"use client"` ——这是一个客户端组件，意味着你可以使用事件监听器和钩子。
- `<input>` - 这是搜索输入。

创建一个新的 `handleSearch` 函数，并在 `<input>` 元素中添加一个 `onChange` 监听器。`onChange` 在输入值变化时会调用 `handleSearch`。

```tsx
// /app/ui/search.tsx

"use client";

import { MagnifyingGlassIcon } from "@heroicons/react/24/outline";

export default function Search({ placeholder }: { placeholder: string }) {
  function handleSearch(term: string) {
    console.log(term);
  }

  return (
    <div className="relative flex flex-1 shrink-0">
      <label htmlFor="search" className="sr-only">
        Search
      </label>
      <input
        className="peer block w-full rounded-md border border-gray-200 py-[9px] pl-10 text-sm outline-2 placeholder:text-gray-500"
        placeholder={placeholder}
        onChange={(e) => {
          handleSearch(e.target.value);
        }}
      />
      <MagnifyingGlassIcon className="absolute left-3 top-1/2 h-[18px] w-[18px] -translate-y-1/2 text-gray-500 peer-focus:text-gray-900" />
    </div>
  );
}
```

通过打开浏览器开发者工具中的控制台，确认它是否正常工作，然后在搜索栏中输入。你应该会看到搜索词被记录在浏览器控制台上。

太好了！你是在捕捉用户的搜索输入。现在，你需要用搜索词更新 URL。

### 2. 用搜索参数更新 URL。

导入 `next/navigation` 的 `useSearchParams` 钩子，并将其赋值为变量：

```tsx
// /app/ui/search.tsx

"use client";

import { MagnifyingGlassIcon } from "@heroicons/react/24/outline";
import { useSearchParams } from "next/navigation";

export default function Search() {
  const searchParams = useSearchParams();

  function handleSearch(term: string) {
    console.log(term);
  }
  // ...
}
```

在 `handleSearch` 中，使用你的新 `searchParams` 变量创建一个新的 [`URLSearchParams`](https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams) 实例。

```tsx
// /app/ui/search.tsx

"use client";

import { MagnifyingGlassIcon } from "@heroicons/react/24/outline";
import { useSearchParams } from "next/navigation";

export default function Search() {
  const searchParams = useSearchParams();

  function handleSearch(term: string) {
    const params = new URLSearchParams(searchParams);
  }
  // ...
}
```

`URLSearchParams` 是一个 Web API，提供操作 URL 查询参数的工具方法。你不用创建复杂的字符串文字，可以用它来获取参数字符串，比如 `?page=1&query=a`。

接下来，根据用户输入设置参数字符串。如果输入是空的，你要删除它：

```tsx
// /app/ui/search.tsx

"use client";

import { MagnifyingGlassIcon } from "@heroicons/react/24/outline";
import { useSearchParams } from "next/navigation";

export default function Search() {
  const searchParams = useSearchParams();

  function handleSearch(term: string) {
    const params = new URLSearchParams(searchParams);
    if (term) {
      params.set("query", term);
    } else {
      params.delete("query");
    }
  }
  // ...
}
```

现在你有了查询字符串。你可以用 Next.js 的 `useRouter` 和 `usePathname` 钩子来更新 URL。

从 `'next/navigation'` 导入 `useRouter` 和 `usePathname`，然后在 `handleSearch` 中使用 `useRouter()` 的替换方法：

```tsx
// /app/ui/search.tsx

"use client";

import { MagnifyingGlassIcon } from "@heroicons/react/24/outline";
import { useSearchParams, usePathname, useRouter } from "next/navigation";

export default function Search() {
  const searchParams = useSearchParams();
  const pathname = usePathname();
  const { replace } = useRouter();

  function handleSearch(term: string) {
    const params = new URLSearchParams(searchParams);
    if (term) {
      params.set("query", term);
    } else {
      params.delete("query");
    }
    replace(`${pathname}?${params.toString()}`);
  }
}
```

以下是目前情况的详细说明：

- `${pathname}` 是当前路径，在你的情况下是 `"/dashboard/invoices"`。
- 当用户在搜索栏输入时，`params.toString()` 会将这些输入转换为友好的 URL 格式。
- 用用户的搜索数据 `replace(${pathname}?${params.toString()})` 更新 URL。例如，如果用户搜索 "Lee"，则 `/dashboard/invoices?query=lee`。
- 得益于 Next.js 客户端导航（你在[页面间导航章节](./第五章.md)中学过），URL 可以不重新加载页面即可更新。

### 3. 保持 URL 和输入同步

为了确保输入字段与 URL 同步并在分享时被填充，您可以通过从 `searchParams` 中读取来传递一个 `defaultValue` 给输入：

```tsx
// /app/ui/search.tsx

<input
  className="peer block w-full rounded-md border border-gray-200 py-[9px] pl-10 text-sm outline-2 placeholder:text-gray-500"
  placeholder={placeholder}
  onChange={(e) => {
    handleSearch(e.target.value);
  }}
  defaultValue={searchParams.get("query")?.toString()}
/>
```

> defaultValue 与 value / 受控 与 非受控
>
> - 如果你用状态管理输入值，你会用值属性将其设为受控组件。这意味着 React 会管理输入的状态。
>   不过，既然你没有使用状态，可以使用 `defaultValue`。这意味着原生输入会管理自己的状态。这样没关系，因为你把搜索查询保存在 URL 而不是状态里。

### 4. 更新表格

最后，你需要更新表组件以反映搜索查询。

返回 invoices 页面。

页面组件[接受一个叫 `searchParams` 的 props](https://nextjs.org/docs/app/api-reference/file-conventions/page) ，所以你可以把当前的 URL 参数传递给 `<Table>` 组件。

```tsx
// /app/dashboard/invoices/page.tsx

import Pagination from "@/app/ui/invoices/pagination";
import Search from "@/app/ui/search";
import Table from "@/app/ui/invoices/table";
import { CreateInvoice } from "@/app/ui/invoices/buttons";
import { lusitana } from "@/app/ui/fonts";
import { Suspense } from "react";
import { InvoicesTableSkeleton } from "@/app/ui/skeletons";

export default async function Page(props: {
  searchParams?: Promise<{
    query?: string;
    page?: string;
  }>;
}) {
  const searchParams = await props.searchParams;
  const query = searchParams?.query || "";
  const currentPage = Number(searchParams?.page) || 1;

  return (
    <div className="w-full">
      <div className="flex w-full items-center justify-between">
        <h1 className={`${lusitana.className} text-2xl`}>Invoices</h1>
      </div>
      <div className="mt-4 flex items-center justify-between gap-2 md:mt-8">
        <Search placeholder="Search invoices..." />
        <CreateInvoice />
      </div>
      <Suspense key={query + currentPage} fallback={<InvoicesTableSkeleton />}>
        <Table query={query} currentPage={currentPage} />
      </Suspense>
      <div className="mt-5 flex w-full justify-center">{/* <Pagination totalPages={totalPages} /> */}</div>
    </div>
  );
}
```

如果你导航到 `<Table>` 组件，你会看到两个 props `query` 和 `currentPage` 都传递给了 `fetchFilteredInvoices()` 函数返回与查询匹配的 invoices。

```tsx
// /app/ui/invoices/table.tsx

// ...
export default async function InvoicesTable({ query, currentPage }: { query: string; currentPage: number }) {
  const invoices = await fetchFilteredInvoices(query, currentPage);
  // ...
}
```

有了这些变化，不妨试试看。如果你搜索一个词，你会更新 URL，这会向服务器发送新的请求，服务器上会获取数据，只有与查询匹配的发票会被返回。

> 什么时候使用 `useSearchParams()` 钩子，什么时候用 `searchParams` 这个 prop？
>
> 你可能注意到你用了两种不同的方法提取搜索参数。你用哪种方式取决于你是在客户端还是服务器端工作。
>
> - `<Search>` 是客户端组件，所以你用 `useSearchParams()` 钩子从客户端访问参数。
> - `<Table>` 是一个服务器组件，它会获取自身数据，所以你可以把 `searchParams` prop 从页面传递给组件。
>
> 一般来说，如果你想从客户端读取参数，可以使用 `useSearchParams()` 钩子，这样可以避免回到服务器。

## 最佳实践：防抖

恭喜你！你已经实现了搜索功能 Next.js！但你可以做一些优化的方法。

在你的 `handleSearch` 功能中，添加以下内容 `console.log`：

```tsx
// /app/ui/search.tsx

function handleSearch(term: string) {

  const params = new URLSearchParams(searchParams);

```

然后在搜索栏输入 "Delba"，在开发工具里查看控制台。发生了什么？

```shell
# Dev Tools Console  开发工具控制台

Searching... D
Searching... De
Searching... Del
Searching... Delb
Searching... Delba
```

你在每一次按键时更新 URL，因此每一次按键都在查询你的数据库！这不是问题，因为我们的应用很小，但想象一下，如果你的应用有成千上万的用户，每个人在每次按键时向数据库发送新的请求。

防抖是一种编程实践，限制函数的触发速率。在我们的情况下，只有当用户停止输入时，你才需要查询数据库。

> 防抖的运作方式
>
> 1. 触发事件 ：当出现应被取消的事件（如搜索框中的按键）时，计时器开始。
> 2. 等待 ：如果在计时器到期前发生新事件，计时器会被重置。
> 3. 执行： 如果计时器倒计时结束，则执行防抖函数。

你可以用几种方式实现防抖，包括手动创建自己的防抖功能。为了简化操作，我们将使用一个叫 [use-debounce](https://www.npmjs.com/package/use-debounce) 的库。

安装 `use-debounce`：

```bash
pnpm i use-debounce
```

在你的 `<Search>` 组件中，导入一个叫 `useDebouncedCallback` 的函数：

```tsx
// /app/ui/search.tsx

// ...
import { useDebouncedCallback } from "use-debounce";

// Inside the Search Component...
const handleSearch = useDebouncedCallback((term) => {
  console.log(`Searching... ${term}`);

  const params = new URLSearchParams(searchParams);
  if (term) {
    params.set("query", term);
  } else {
    params.delete("query");
  }
  replace(`${pathname}?${params.toString()}`);
}, 300);
```

该函数会包裹 `handleSearch` 的内容，只有在用户停止输入后（300 毫秒）后才运行代码。

现在再输入搜索栏，然后在开发工具里打开控制台。你应该会看到以下内容：

```shell
# Dev Tools Console  开发工具控制台

Searching... Delba
```

通过 防抖，你可以减少发送到数据库的请求数量，从而节省资源。

## 添加分页

引入搜索功能后，你会注意到表格一次只显示 6 张 invoices 。这是因为 `data.ts` 中的 `fetchFilteredInvoices()` 函数每页最多返回 6 张 invoices。

添加分页功能允许用户在不同页面中浏览，查看所有 invoices 。让我们看看你如何像搜索一样，利用 URL 参数实现分页。

进入 `<Pagination/>` 组件，你会发现它是一个客户端组件。你不想在客户端获取数据，因为那样会暴露数据库秘密（记住，你没有用 API 层）。相反，你可以在服务器上获取数据，然后作为一个道具传递给组件。

在 `/dashboard/invoices/page.tsx` 中，导入一个名为 `fetchInvoicesPages` 的新函数，并将 `searchParams` 的查询作为参数传递：

```tsx
// /app/dashboard/invoices/page.tsx

// ...
import { fetchInvoicesPages } from '@/app/lib/data';

export default async function Page(
  props: {
    searchParams?: Promise<{
      query?: string;
      page?: string;
    }>;
  }
) {
  const searchParams = await props.searchParams;
  const query = searchParams?.query || '';
  const currentPage = Number(searchParams?.page) || 1;
  const totalPages = await fetchInvoicesPages(query);

  return (
    // ...
  );
}
```

f`etchInvoicesPages` 返回基于搜索查询的总页数。例如，如果有 12 张与搜索查询匹配的 invoices ，每页显示 6 张 invoices ，那么总页数为 2 页。

接下来，将 `totalPages` prop 传递给 `<Pagination/>` 组件：

```tsx
// /app/dashboard/invoices/page.tsx

// ...

export default async function Page(props: {
  searchParams?: Promise<{
    query?: string;
    page?: string;
  }>;
}) {
  const searchParams = await props.searchParams;
  const query = searchParams?.query || "";
  const currentPage = Number(searchParams?.page) || 1;
  const totalPages = await fetchInvoicesPages(query);

  return (
    <div className="w-full">
      <div className="flex w-full items-center justify-between">
        <h1 className={`${lusitana.className} text-2xl`}>Invoices</h1>
      </div>
      <div className="mt-4 flex items-center justify-between gap-2 md:mt-8">
        <Search placeholder="Search invoices..." />
        <CreateInvoice />
      </div>
      <Suspense key={query + currentPage} fallback={<InvoicesTableSkeleton />}>
        <Table query={query} currentPage={currentPage} />
      </Suspense>
      <div className="mt-5 flex w-full justify-center">
        <Pagination totalPages={totalPages} />
      </div>
    </div>
  );
}
```

导航到 `<Pagination/>` 组件，导入 `usePathname` 和 `useSearchParams` 的钩子。我们会用它获取当前页面并设置新页面。确保也取消该组件中的代码注释。你的应用程序会暂时崩溃，因为你还没有实现 `<Pagination/>` 逻辑。现在就这么做！

接下来，在 `<Pagination>` 组件中创建一个名为 `createPageURL` 的新函数。和搜索类似，你用 `URLSearchParams` 设置新的页码，用 `pathName` 创建 URL 字符串。

```tsx
// /app/ui/invoices/pagination.tsx

"use client";

import { ArrowLeftIcon, ArrowRightIcon } from "@heroicons/react/24/outline";
import clsx from "clsx";
import Link from "next/link";
import { generatePagination } from "@/app/lib/utils";
import { usePathname, useSearchParams } from "next/navigation";

export default function Pagination({ totalPages }: { totalPages: number }) {
  const pathname = usePathname();
  const searchParams = useSearchParams();
  const currentPage = Number(searchParams.get("page")) || 1;

  const createPageURL = (pageNumber: number | string) => {
    const params = new URLSearchParams(searchParams);
    params.set("page", pageNumber.toString());
    return `${pathname}?${params.toString()}`;
  };

  // ...
}
```

以下是目前情况的详细说明：

- `createPageURL` 创建当前搜索参数的实例。
- 然后，它将 "page"参数更新为提供的页码。
- 最后，它利用路径名和更新后的搜索参数构建完整的 URL。

`<Pagination>` 组件的其余部分涉及样式和不同状态（first、last、active、disabled 等）。本课程我们不会详细介绍，但你可以浏览代码，看看 `createPageURL` 是被调用的。

最后，当用户输入新的搜索查询时，你需要将页码重置为 1。你可以通过更新 `<Search>` 组件中的 `handleSearch` 函数来实现：

```tsx
// /app/ui/search.tsx

'use client';

import { MagnifyingGlassIcon } from '@heroicons/react/24/outline';
import { usePathname, useRouter, useSearchParams } from 'next/navigation';
import { useDebouncedCallback } from 'use-debounce';

export default function Search({ placeholder }: { placeholder: string }) {
  const searchParams = useSearchParams();
  const { replace } = useRouter();
  const pathname = usePathname();

  const handleSearch = useDebouncedCallback((term) => {
    const params = new URLSearchParams(searchParams);
    params.set('page', '1');
    if (term) {
      params.set('query', term);
    } else {
      params.delete('query');
    }
    replace(`${pathname}?${params.toString()}`);
  }, 300);

```
