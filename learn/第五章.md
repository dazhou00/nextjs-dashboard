# 页面间导航

在上一章中，你创建了 dashboard 的布局和页面。现在，让我们添加一些链接，以便用户在dashboard 个路由之间进行导航。

- 如何使用 `next/link` 组件。
- 如何使用 `usePathname()` 钩子显示活动链接。
- Next.js 中的导航是如何工作的。

## 为什么要优化导航？

要在页面之间建立链接，传统上会使用 `<a>` HTML元素。目前，侧边栏链接使用的是 `<a>` 元素，但请注意，当你在浏览器中在首页、发票页和客户页之间导航时会发生什么。

你看到了吗？

每次页面导航都会有整页刷新！

## `<Link>` 组件

在 Next.js 中，你可以使用 `<Link />` 组件在应用程序的各个页面之间进行链接。 `<Link>` 允许你通过 JavaScript 进行 [客户端导航](https://nextjs.org/docs/app/getting-started/linking-and-navigating#how-routing-and-navigation-works)。

要使用 `<Link />` 组件，请打开 `/app/ui/dashboard/nav-links.tsx` ，并从 `next/link` 导入 `Link` 组件。然后，将 `<a>` 标签替换为 `<Link>`：

```tsx
// /app/ui/dashboard/nav-links.tsx

import { UserGroupIcon, HomeIcon, DocumentDuplicateIcon } from "@heroicons/react/24/outline";
import Link from "next/link";

// ...

export default function NavLinks() {
  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <Link
            key={link.name}
            href={link.href}
            className="flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3"
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </Link>
        );
      })}
    </>
  );
}
```

如你所见，`Link` 组件与使用 `<a>` 标签类似，但它不是用 `<a href="…">` ，而是用 `<Link href="…">` 。

保存你的更改，然后检查它在你的本地主机上是否能正常运行。现在，你应该能够在页面之间导航，而不会看到完全刷新的情况。尽管应用程序的部分内容是在服务器上渲染的，但不会有整个页面的刷新，这使其感觉起来像一个原生的网络应用。这是为什么呢？

## 自动代码拆分和预加载

为了改善导航体验，Next.js 会按路由段自动对应用程序进行代码分割。这与传统的 React 单页应用（[SPA](https://nextjs.org/docs/app/guides/single-page-applications)）不同，在传统的 React 单页应用中，浏览器会在初始页面加载时加载所有的应用程序代码。

按路由拆分代码意味着页面会变得相互独立。如果某个页面抛出错误，应用程序的其余部分仍能正常运行。这也减少了浏览器需要解析的代码量，从而使你的应用程序运行得更快。

此外，在生产环境中，只要 `<Link>` 组件出现在浏览器的视口中，Next.js 就会在后台自动预获取链接路由的代码。当用户点击该链接时，目标页面的代码早已在后台加载完成，这正是页面切换几乎能瞬间完成的原因！

了解更多关于[导航的工作原理](https://nextjs.org/docs/app/getting-started/linking-and-navigating#how-routing-and-navigation-works)。

## 模式：显示活动链接

一种常见的用户界面模式是显示一个活动链接，向用户表明他们当前所在的页面。要实现这一点，你需要从 URL 中获取用户的当前路径。Next.js 提供了一个名为 [`usePathname()`](https://nextjs.org/docs/app/api-reference/functions/use-pathname) 的钩子，你可以用它来检查路径并实现这种模式。

由于 [`usePathname()`](https://nextjs.org/docs/app/api-reference/functions/use-pathname) 是一个React钩子，你需要将 `nav-links.tsx` 转换为客户端组件。在文件顶部添加React的 `"use client"` 指令，然后从 `next/navigation` 导入 `usePathname()`：

```tsx
// /app/ui/dashboard/nav-links.tsx

"use client";

import { UserGroupIcon, HomeIcon, DocumentDuplicateIcon } from "@heroicons/react/24/outline";
import Link from "next/link";
import { usePathname } from "next/navigation";

// ...
```

接下来，在你的 `<NavLinks />` 组件内部，将路径赋值给一个名为 `pathname` 的变量：

```tsx
// /app/ui/dashboard/nav-links.tsx
export default function NavLinks() {
  const pathname = usePathname();
  // ...
}
```

> 注意：`nav-links.tsx` 并不是 Next.js 的特殊文件——你可以给它起任何名字。如果你重命名了它，请确保相应地更新导入语句。

你可以使用 [CSS样式](./第二章.md) 章节中介绍的 `clsx` 库，在链接处于活跃状态时，有条件地应用类名。当 `link.href` 与路径名匹配时，该链接应显示为蓝色文本和浅蓝色背景。

这是 `nav-links.tsx` 的最终代码：

```tsx
// /app/ui/dashboard/nav-links.tsx

"use client";

import { UserGroupIcon, HomeIcon, DocumentDuplicateIcon } from "@heroicons/react/24/outline";
import Link from "next/link";
import { usePathname } from "next/navigation";
import clsx from "clsx";

// ...

export default function NavLinks() {
  const pathname = usePathname();

  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <Link
            key={link.name}
            href={link.href}
            className={clsx(
              "flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3",
              {
                "bg-sky-100 text-blue-600": pathname === link.href,
              },
            )}
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </Link>
        );
      })}
    </>
  );
}
```

保存并检查你的本地主机。现在你应该能看到活动链接以蓝色高亮显示。
